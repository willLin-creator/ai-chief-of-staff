Lead with the recommendation, then support it. Cut hedging and filler. Prefer short
sentences and structure over prose when structure is clearer.

**Why:** They optimize for fast, high-quality decisions and read a lot of drafts.

**How to apply:** In any draft or response, put the answer first. If a framework
needs a lot of setup to land, the recommendation needs work, not more setup. See
also [[never-send-without-approval]].
